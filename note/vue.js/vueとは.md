## vue.jsとは
・JavaScriptのフレームワーク
・学習コストが低く、ライブラリやツールが豊富
・ウェブページが速くサクサク動く
・何かが変わるとすぐに自動的にその変化が表示される
・運用保守を意識して開発される。
```
* Reactしているけど一番の違いは運用保守の点か？
* 他は割と近いイメージもある
* フレームワークっていう点で振れやすい部分ではあるのかな
```

## 特徴
* シンプルな開発工程、高速な動作
DOMを仮想化した「仮想DOM」を使うことで処理を高速にしている
シンプルかつ高速というアプリ開発にはこれ以上ない条件

* 依存関係のないコンポーネント
Reactでもやってきたが、ファイルを部品にすることで依存関係を少なくする。
共通部品も可

* コンパクトなファイルサイズ
gzip圧縮で20kbほどにできる
5000～10000時くらいのファイルサイズと同じ

* アプリケーションの仕様変更に対して柔軟に対応できる
コンポーネントベースのアーキテクチャ

* データの同期を自動化⇒画面に反映
データの同期により双方向にバインディングし、モデルデータと画面表示を自動的に同期する

## vue.jsでできること
* イベント処理が得意
ライフサイクルフックというイベント管理用機能が充実している

* マウスカーソルの動きを視覚化できる
ユーザーのマウス操作をトレースした線を描く

クリックした場所の色を変更し、徐々に元に戻る

といったマウスの動きを可視化する

* 入力した値をリアルタイム反映
Vue.jsでは入力した値を即座に表示に反映・連動させるリアクティブな仕組みを構築できる

例、「体重と身長を入力すると即座にBMI値が表示されるBMI計算機」

「入力した文字に特定の符号（濁点など）をつける機能」

* 図やチャートの描画
「アニメーションのように連続して滑らかにチャートを描く」など動的な表現ができる

* SPAの構築
SPAとは「単一のページとして動作するアプリケーション」の総称
ページ遷移が発生しないため通信負荷が小さく、外観の同一性も維持しやすいため、クロスプラットフォーム対応のアプリケーションなどで頻繁に使用されている

画面遷移を最小限におさえることでユーザーのストレスを減らしつつ、トップページの情報量やインパクトを高めることでUX（ユーザー体験）を高められるメリットもある

ゼロからSPAを作るとなると、JavaScriptによる大量のコーディング作業が必要

Vue.jsは独立したコンポーネント（ヘッダーメニューなど複数ページで使い回すパーツ）の組合せによって素早くアプリケーションを構築できるため、SPAの構築に適したフレームワークといえる


## Vue.jsを使うメリット
１．学習コストが低い
２．HTML内で手軽に使える
３．DOM操作が自動化
４．TypeScriptを全面サポート
５．シンプルrで拡張性が高い

## Vue.jsを使うデメリット
### １．大規模開発には不向き
・Vue.jsのデータ構造だと、大規模アプリのデータ管理が煩雑になる

・コンポーネントが多いページだと動作が重くなる可能性がある

リプレイスではない限りReactを採用するケースが多い

### ２．ネイティブアプリ開発用としては発展途上
たとえばReactの場合、React Nativeと呼ばれるWebアプリケーション開発と同様の文法でiOS/Androidアプリを開発するフレームワークが存在
