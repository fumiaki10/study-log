# どうやってカウントダウンさせるか

## isActive
これはいわゆるON・OFFスイッチ
ボタンを押してtrueかfalseに変えることで
ボタンを減らすかどうか条件分岐で書いている

## 依存配列にsetCountTimerを入れていた間違い
更新関数（set～）はReactの仕組みで「絶対に値が変わらない不変のもの」として扱われるため、変化したとみなされない。

## 0になっても減り続ける不具合
```tsx
 useEffect(() => {
    const intervalId = setInterval(() => {
      if (isActive === true) {
        setCountTimer((prevCount) => prevCount - 1);
      }
      if (countTimer === 0) {
        setIsActive(false)
      }
    }, 1000);

    return () => clearInterval(intervalId);
  }, [isActive, countTimer])
```
このコードだとだめだった

## 改善案
条件式を最初に持ってきて、0秒かつtrueのときsetIsActiveがfalseになるように追記
```tsx
 useEffect(() => {
    if (countTimer <= 0 && isActive === true) {
      setIsActive(false)
    } else {
      const intervalId = setInterval(() => {
        if (isActive === true) {
          setCountTimer((prevCount) => prevCount - 1);
        }
      }, 1000);

      return () => clearInterval(intervalId);
    }
  }, [isActive, countTimer])
```

## 更にすっきりさせた部分
```tsx
import { useEffect, useState } from 'react'
import '../assets/Timer.css'

function Timer() {

  const [countTimer, setCountTimer] = useState(0)
  const [isActive, setIsActive] = useState(false)

  useEffect(() => {
    const nextTimer = countTimer - 1 //特に今必要ではなかった
    if (countTimer <= 0 && isActive === true) {
      setIsActive(false)
    } else {
      const intervalId = setInterval(() => {
        if (isActive === true) {
          setCountTimer(nextTimer); //わざわざ関数にしなくてもよかった部分
        }
        if (countTimer === 0) {  //冒頭のifで0になったときfalseになるよう作っているのでここの処理は不要
          setIsActive(false)
        }
      }, 1000);

      return () => clearInterval(intervalId);
    }
  }, [isActive, countTimer])
```

## カウントを手動でもマイナスにできないように
```tsx
 <button className='timer-button' onClick={() => setCountTimer(countTimer - 1)}>-1秒</button>
```
 のままだと0秒の時ボタンを押してマイナスにできてしまうので

```tsx
  <button className='timer-button' onClick={() => setCountTimer(countTimer > 0 ? countTimer - 1 : 0)}>-1秒</button>
```
といった感じで条件式を入れる

* 別解
Math.max(-5,0) //0が返ってくるようにもなる
とも書ける

なので
```tsx
 onClick={() => setCountTimer(Math.max(countTimer - 1, 0))}>

```
という別解がある


## タイムラグが出る
カウントダウン中に時間を足したり引いたりすると
変な間が生まれる
⇒useEffectを再読み込みして一秒の周期をやり直しているせい
```tsx
 <button className='timer-button' onClick={() => setCountTimer(countTimer + 1)}>+1秒</button>

```
```tsx
 <button className='timer-button' onClick={() => setCountTimer((prev) => prev + 1)}>+1秒</button>
 ```
 にすることでとりあえず前の値に足しておいてという形になった

 * このあたりで秒数を足し続けたりすると数字が変化しない事態になる
 理由⇒CountTimerを監視対象にしていたので、数字が変わったときsetCountTimerで動かしているのでuseEffectの対象になってしまった
 ⇓
 なので監視対象から外した[isActive]だけにする


 ## 追記
 countTimer - countTimer
 引きたい数字以下の時
 59秒のときに60で引きたいとき
```tsx
  <button className='timer-button' onClick={() => setCountTimer((prev) => prev > 60 ? prev - 60 : countTimer - countTimer)}>-1分</button>
```
みたいに記述して、0になるようにしたかったけど

依存配列からcountTimerを消したため、過去の数時（固定になってしまう）ので
0とそのまま書いて0にする

3600　⇒　start　⇒　－3600みたいにやるとマイナスになってしまう
trueになっているときにcountTimer <= 0になっているからここでfalseが働くと思ったんだけど

0に手動でした瞬間
isActive(true)のままになっている
変化していないから

## ジレンマ
countTimerを監視対象にするとuseEffectが最初から働いてカウントがずれる
入れないとボタンが対象にないので、isActiveが条件式にそもそもあたらない

## * A 条件分岐で解決した
```tsx
  useEffect(() => {
    const intervalId = setInterval(() => {
      if (isActive === true) {
        setCountTimer((prevCount) => {
          if (prevCount <= 1) { //1以下になったときそこでストップかけられるようにした。return 0にしてストップをかける
            setIsActive(false);
            return 0
          }
          return prevCount - 1 //数字が1以上ならいつも通りの処理
        });
      }
    }, 1000);

    return () => clearInterval(intervalId);

  }, [isActive])

```
