## 1.メインファイルの切り替え
* 初期からあるApp.tsxではなく、自作したrouterApp.tsxをメイン画面にしたかったけど切り替え方がわからなかった

**解決法**
アプリのエントリーポイントで（最初に読み込まれる場所）あるsrc/main.tsxの中身のAppのインポート先と呼び出しタグを<RouterApp/>へ書き換えた
```tsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import RouterApp from './routerApp.tsx'

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <RouterApp />
  </StrictMode>,
)

```

同時に、Reactの「コンポーネント名は必ず大文字から始める」というルールに従い、関数名をrouterAppからRouterAppへ修正

## 2.コンポーネントの重複宣言エラー
* routerApp.tsxで画面（Home、Page1～3）をインポートする際、すべて{Home}という同じ名前で読み込もうとしていた
```tsx
import { Home } from './components/react-router/Home.jsx'
import { Home } from './components/react-router/Page1.jsx'
import { Home } from './components/react-router/Page2.jsx'
import { Home } from './components/react-router/Page3.jsx'
```

```
Identifier 'Home' has already been declared //既に宣言されている
```
既に宣言されているというエラーが出た

解決方法
インポート文を修正
```tsx
import { Home } from './components/react-router/Home.jsx'
import { Page1 } from './components/react-router/Page1.jsx' //  Page1 に変更
import { Page2 } from './components/react-router/Page2.jsx' //  Page2 に変更
import { Page3 } from './components/react-router/Page3.jsx' //  Page3 に変更
```

## 3.React Routerのバージョンによる記述エラー
* バージョンの古い宣言を使っていた（バージョン5）
```
<Switch>
<exact>
```
最新のバージョン６では
* <Switch> => <Routes>
* <exact> => 自動化されたため削除された

**画面の指定方法**
```tsx
<Routes Path="..." element={<コンポーネント />}/>
```

## 4.TypeScriptの型エラーとエディタのフリーズ
TypeScriptからjsxのファイルを読み込もうとしたため、暗黙的にany型になりますという型エラーが発生した

**解決法**
* 対象ファイルの拡張子の変更
Home.jsx => Home.tsx
* エディタが古いファイル情報を記憶して固まっていたため、コマンドパレットから「TypeScriptサーバーを再起動する」を実行して、エディタエラー表示を正常に反映した
