## useNavigateとは
* React Routerが提供するフック
* イベントハンドラや副作用の内部で、任意のタイミングでページ遷移（ナビゲーション）を行う関数を返す
* プログラム（JavaScriptの処理）を使って、好きなタイミングで画面を切り替えるための道具
```tsx
import { useNavigate } from 'react-router-dom';

function Home() {
  const navigate = useNavigate();

  const goToAbout = () => {
    navigate('/about');
  };

  return <button onClick={goToAbout}>Go to About</button>;
}

export default Home;
```
ボタンが押された（イベント）⇒画面を遷移

### なぜこれを使うのか
```
通常リンク<a>や<Link>があるから使わないのでは？
```
となるが、
**「処理が終わった後画面を切り替えたい」**
という場面が出てくる

* フォーム送信後：「保存ボタン」⇒サーバーへのデータ保存が成功⇒一覧に戻る
* ログイン成否：パスワード認証⇒自動的にトップページ
* 自動リダイレクト：３秒待ったら自動的にトップページ

**ロジック＋画面遷移をセットで使いたい**ときに使える

## 裏で起きていること
navigate('/about')が実行された時、ReactRouterの内部ではブラウザの「History API」という仕組みが動く

* 履歴の追加：ブラウザの目に見えない履歴帳に /about という1ページを新しく書き加える
* URLの書き換え：ページ全体をリロード（再読み込み）することなく、アドレスバーのURLだけを /about に書き換える
* コンテナへの通知：「ルートをまとめるコンテナ」に対して「URLが変わったよ！」と合図を送る

これにより、コンテナが /about に対応するコンポーネントを探して画面をパッと切り替わる

## 相対的な移動（履歴の操作）
navigate関数には、URLを渡す以外に

**「数字を渡す」**

という重要な役割がある

* navigate(-1):
ブラウザの「戻る」ボタンをプログラムで押したのと同じ状態（1つ前のページに戻る）になる
* navigate(-2): 2つ前のページに戻る

わざわざ「前のページのURL」を調べて指定しなくても、ユーザーの移動履歴を元に「戻す・進める」ができるため、「戻るボタン」や「キャンセルボタン」を自作する際に必須の概念となる
