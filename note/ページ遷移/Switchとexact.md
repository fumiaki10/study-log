## Switchとは
* **「上から順番にURLをチェックして、最初に一致した1つの画面だけを表示する」ための仕切り役**もし Switch がないと、URLに少しでも一致するパーツがある場合、複数の画面が同時に1つのページに縦並びで表示されてしまうという問題があった。

* Switchがあるとき： 1つの画面だけを表示
* Switchがないとき： 条件に合う画面を全部表示（バグの原因になる）

そのため、昔は「ここからここは画面を切り替えるエリアだよ」と伝えるために、全体を 
```
<Switch>〜</Switch>
```
 で囲む必要があった。

※新バージョンの <Routes> では、最初から「必ず1つだけを表示する」という仕組みに強化されたため、Switch は役割を終えて廃止された
## exact とは？
* **「URLが完全に一致したときだけ、その画面を表示してね」と指定するためのスイッチ（プロパティ）**

英語の「exact（正確な、厳密な）」という意味そのままで、昔の React Router は前方一致（URLの先頭部分だけが合っていればOK）という大雑把な判定をしていた。

例えば、以下のような設定があったとします。
```
/ ➔ ホーム画面/
page1 ➔ ページ1画面
```

もし exact をつけていないと、ユーザーが /page1 にアクセスしたとき、

React Router は「お、先頭に / が含まれてるからホーム画面を表示しよう！」

と勘違いしてしまい、いつまで経ってもページ1にたどり着けない（ホーム画面が表示されてしまう）という現象が起きていた。

これを防ぐために、「前方一致じゃなくて、完全にURLが一致したときだけ表示して！」と命令するのが exact 。

※新バージョンでは、最初から「完全一致」で賢く判定してくれるようになったため、わざわざ exact と書く必要がなった。
