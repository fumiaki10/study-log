## 今までのコード
```jsx
function TextChange(props) {
  return (
    <div>
      <button onClick={() => props.onSendMessage(props.SendText)}>
        {props.text}
      </button>
    </div >
  );
}

export default TextChange
```

## 分割代入をしたコード
```jsx
function TextChange({text,sendText,onSendMessage}){
  return(
    <button onClick={() => onSendMessage(sendText)}>
      {text}
    </button>
  )
}

```

**POINT**
* 引数の部分：propsだった部分に({text,sendText,onSendMessage})を入れ書き換えた
* 中身のアイテムをそのまま使えるようにした

## この書きかたの利点
* 単純に見やすい
* このコンポーネントは、「親から何のデータを求めているのか」が、関数の1行目でわかる

## よくある間違い
(text,sendText,onSendMessage)にしていたが間違い

({text,sendText,onSendMessage})だった

## 暗黙的にanyになります
* TypeScriptを使っているときに出る特有のエラー
ファイル名がjsxではなくtsxになっていると出る
**解決法**
親から送られてくるデータの型があらかじめ指定されていないといけないので
```tsx
interface TextChangeProps{
  text:string;
  SendText:string;
  onSendMessage:(data:string) => void;
}

function TextChange({ text, SendText, onSendMessage }: TextChangeProps) {
```
