## em要素
* イタリックの斜体に文字を変換できる
* ただ斜体にするのであればfont-styleで調整するが、意味的強調の形を含めて表記することがある
* css単位でもemはある。文字の高さを基準に指定できる長さの単位
* 音声読み上げによってトーンを強調してくれる
* <i>もイタリックにしてくれる


個人サイトではあまり使われない表現も多いが、大規模ウェブサイト、ニュースサイト、企業の公式サイトになるとしっかり分ける必要がある

```
1. strong（強い重要性）
意味：最も重要。警告や、重要なキーワード。
見た目：太字になります。
例文：「この薬は、食後30分以内に服用してください。」
理由：飲み忘れると危険なため、読者の注意を強く惹きつけます。

2. em（文章内の強調）
意味：文脈のアクセント。読み上げソフトでは声のトーンが高くなります。
見た目：斜体になります。
例文：「彼はただの友達ではなく、親友です。」
理由：どの部分を強調して読ませたいかで使い分けます。

3. cite（作品名・引用元のタイトル）
意味：書籍、映画、論文、歌などの「作品タイトル」や出典。
見た目：斜体になります。
例文：「宮崎駿監督の映画『となりのトトロ』は有名です。」
理由：単なる単語ではなく、特定の「創作物」の名称であることを明確に定義します。

4. i（他と区別される文字列）
意味：周囲のテキストとは性質が異なる文字。外来語、専門用語の定義、架空のキャラクターの心の声など。
見た目：斜体になります。
例文：「彼はサムシング・ニューを求めて旅に出た。」
理由：「サムシング・ニュー」というカタカナ語（外来語）であることを明確に区別します。
```

## URI
* Uniform Resource Identifier 統一資源識別子
* IDの語源 Identifier
* URLとURNを含んだ言葉
### 識別子
複数の対象から、特定の一つを区別するために利用する名前や数字

ファイル名、ドライブ名、変数、関数

例、example1,example2
### URL
Uniform Resource Locator　統一資源位置指定子

Locator => 位置、住所

スキーム、ホスト、パスといった書き方のルールも決まっている
```
例
【http://rtarou:abcd@www.rstone-jp.com:12345/column/index.htm?jouhou=python+backend#table01】
```
スキーム:httpの部分
オーソリティ:http:// この//以降から始まる部分
ユーザー情報:ウェブページを利用するユーザー名とパスワード。rtarou:abcdに相当する
ホスト：ウェブページの情報が格納されているサーバー名。www.rstone.comの部分
ポート:サーバーにあるポート番号。:12345の部分
* ポート番号とは、通信プロトコルの一種「TCP/IP」での通信先を決めるための番号。
パス：オーソリティ内の細かい利用先のこと。column/index.htmlの部分
クエリ:パス内のさらに細かい利用先。jouhou=python+backendの部分
フラグメント:http:ではWeb終え0時内の特定の情報を指定するためのもの。#table01の部分。URIの末尾に#を着けて指定。ブラウザで処理が実行されサーバー側にデータが送信されない特徴がある
### URN
Uniform Resource Name 統一資源名
* URLが住所
* URNがウェブページの名前
* 【urn:abcd:12345】のような表記
abcdの部分が「名前空間の識別子」　区切りみたいなもの

一般にURLが表示されているが、URNはあくまでインターネット上での識別子。サーバー移行でＵＲＬが変更することはあるがＵＲＮは基本的に変更はない
