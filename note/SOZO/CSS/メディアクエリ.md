## メディアクエリとは
cssで設定できる、画面の幅によってcssを設定できる

##　書きかた
@media media-type and(media-feature-rule){
  cssのルールをここに追加
}
* media-type:このコードがどのようなメディアのためのものであるかをブラウザーに伝える
```
all（すべて）
print（印刷）
screen（画面）
```
```css
@media print {
  body {
    font-size: 12pt;
  }
}
```
ページ印刷されるときのみ、body12pxが働く。ブラウザー読み込みに適用されない

* media-feature-rule:ルールを使用してメディア特性を対象にできる
```css
@media screen and (width: 600px) {
  body {
    color: red;
  }
}
```
正確に600pxの幅の時bodyの文字の色が赤になる
```css
@media screen and (max-width: 600px) {
  body {
    color: blue;
  }
}
```
widthが600pxまでbodyの文字の色が青になる

それ以上の幅の時は適用されない

```css
@media (orientation: landscape) {
  body {
    color: rebeccapurple;
  }
}
```
画面の幅が縦長か横長かで色を変える
* landscape => 横長のとき
* portrait => 縦長のとき
上記の例だと横長のとき紫色にしている
```css
@media screen and (hover: hover) {
  body:hover {
    color: white;
    background: black;
  }
}
```
画面にカーソルをのっけた時文字は白に、背景は黒になる


```css
@media (min-width: 30em) and (max-width: 50em) {
  /* … */
}

@media screen and (width >= 600px) and (orientation: landscape) {
  body {
    color: blue;
  }
}

@media screen and (width >= 600px), screen and (orientation: landscape) {
  body {
    color: blue;
  }
}
```
範囲指定ができたり複数指定ができる

##　間違いやすい悩みやすい
* min-width:768px => 768px以上の画面幅に適用
最小値が768
* max-width 768px => 768px以下の画面幅に適用する
最大値が768

* hover:hoverに設定するのはスマホとPCの区別
スマホなどのタッチデバイスでホバーエフェクトを設定　＝＞ユーザーがリンクをタップするとき「1回目でホバー状態、2回目でリンク先に飛ぶ」という状態になることがあるので、これを防ぐ

**hoverが可能な環境（マウス、トラックパッドなど）にだけ**、リッチなアニメーションやツールチップの表示を適用できる

```css
/* PCなど、マウスでホバーできる環境にのみ適用 */
@media (hover: hover) {
  a:hover {
    color: #ff0000;
    text-decoration: underline;
  }
}

```
