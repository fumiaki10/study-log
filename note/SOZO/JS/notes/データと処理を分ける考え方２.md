```js
function showItem(e) { 
  if (e.target.nodeName === 'INPUT') {
     FlexRemoveButtons() 
     if (id === flex_button1) 
     FlexItems.classList.add("class", "btn-primary")

```
だと結局数がいる

## だから押されたとき、その押されたやつが教えてくれるようにする
イベント移譲　＋　e.target

これによって親の監視　＋　見つけた瞬間の情報

これらをまとめて手にできるようにする

```
button3
    ↑
buttons
    ↑
body
    ↑
html
```
こういう親へ親へ伝わるのを
**イベントバブリング**という

## やり方
```html
  <input type="radio" name="button" data-class="btn-primary">
```
ボタンにdata-classを持たせる
### data-class
表示に影響がないデータのこと。この場合特にcssの影響が見た目にあらわれるデータを持っておける

```js
  let flexButton = document.getElementById('buttons');
  flexButton.addEventListener('click', e => showItem(e))
```
ボタンをまとめている親要素を取得⇒クリックしたときのイベント移譲を行う

```js
  function showItem(e) {
    FlexRemoveButtons()
    FlexItems.classList.add(e.target.dataset.class)
  }
```
子に起きたイベントの処理を書く
