## Conceptual Inquiry
HTMLがJSにデータを送る概念
```html
<input type="radio" name="button" onclick=changeHandle("btn-primary") value="1" id="flex_button1">
```
いわゆるこの部分
```html
<input onclick="changeHandle('btn-primary')">
```

設計の概念で**「データ」と「処理」は分ける**

といった考えがある

この記述だとHTMLがその両方を持つことになってしまう

HTMLの役割は「何があるか、どんな値が、ボタンやオブジェクトなんかがあるか」が重要

JSは処理を担う
## やったこと
```js
  function showItem(e) {
    if (e.target.nodeName === 'INPUT') {
      FlexRemoveButtons()
      FlexItems.classList.add(e.target.dataset.class)
    }
  }

  let flexButton = document.getElementById('buttons');
  flexButton.addEventListener('click', e => showItem(e))
```
```html
<input type="radio" data-class="btn-primary">
```
* clickしたときイベント移譲により親要素にまとめる
* htmlのボタンにdata-classを持たせ、処理はjsに
* e.target.dataset.classで押されたボタンのdata-classを呼び出す　＝＞　結果classが付与されることで見た目が変化する


### 補足
showItem(e) このeはイベント全部を表している

ので
```
e.target
e.type
e.target.value
e.timeStamp
e.bubbles
```
これらからe.targetを取り出しているに過ぎない

* これなら直接classを引っ張っているので、getElementByIdで宣言する必要がなくなった

###　流れ
* まず一緒になっている処理とデータ部分を

それぞれ処理、とデータに分ける
```js
ItemButton.addEventListener('click', function () {
    FlexRemoveButtons()
    FlexItems.classList.add("class", "btn-primary")
  })

処理：クリック、削除、追加

データ：付与されるクラス、ボタン
```
* Generation-Then-Comprehension 設計を読み解く
ボタン１　⇒　クラスを消す　⇒　btn-primaryの追加

⇓これを図にすると

クリック　⇒　クラス削除　⇒　クラス追加

⇓実際に

クリック　⇒　関数　⇒　追加するclass名だけ受け取る

**処理は一つ、値だけ変更したい**になる

Parameterization パラメーター化　というらしい
