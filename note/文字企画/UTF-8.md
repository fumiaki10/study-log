## UTF-8とは
* 世界中のほぼすべての文字を表現できる国際的な文字コード企画
* 文字に応じて１～４バイトの可変長でデータを表現
* 半角英数字は１バイト、日本語の漢字やひらがなは主に３バイト

##　主な特徴、仕組み
* 上位互換性：最初の１２８文字（半角英数字や記号）は従来のASCIIコードと完全に一致
* 可変長エンコーディング: 文字によって使用するバイト数が異なり、効率的なデータ保存が可能
```
/ 1バイト: 半角英数字（A, a, 1など）
/ 2バイト: ギリシャ文字やキリル文字など
/ 3バイト: 日本語の漢字、ひらがな、カタカナなど
/ 4バイト: 絵文字や一部の特殊な漢字など
```

## なぜUTF‐８が主流なのか
ウェブサイトの構築からプログラミング、データベースに至るまで、現在最も広く使われている文字コードである。
異なる言語が混在していても文字化けしにくく、インターネットの世界基準となっている。

##　ダウンロードファイルが文字化けした
* きっかけ
どのファイルをダウンロードしても、exeのタイトルやtxtのタイトルと中身などが文字化けする

**解決法**
設定⇒時刻と言語⇒言語と地域⇒Windowsの標準言語の↓で詳細表示⇒ベータ版：世界中の言語に対応するためにUnicode UTF-8を使用します⇒オフにする

## 何故文字化けしたのか
* 日本語の「方言」の違い
日本のパソコン（特に古いアプリやテキストファイル）は、昔から 「Shift-JIS」 という日本独自の文字ルールを使っている。

一方で、世界共通の最新ルールが 「UTF-8」 。チェックボックスのチェック（世界中の言語サポート…）は、「Windowsの全アプリを、強制的に最新の世界共通ルール（UTF-8）で動かす」という実験的な設定であった。

翻訳ミスでパニックにこのチェックが入ると、Windowsは「Shift-JIS」で書かれたアプリ（exe）やテキスト（txt）の名前や中身を、無理やり「UTF-8」の辞書で読もうとする。

結果、言語の翻訳がめちゃくちゃになり、画面上が「縺ゅ＞縺...」のような宇宙人の言葉（文字化け）になってしまう。
